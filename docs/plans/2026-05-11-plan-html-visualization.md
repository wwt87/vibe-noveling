# novel-plan HTML 可视化

**日期**：2026-05-11

## 问题

`novel-plan` 产出的 `故事梗概.md` 和 `大纲.md` 是纯文本 Markdown，内容量大，用户在终端中难以快速审阅。需要更直观的可视化方式。

## 决策

用 HTML 可视化替代这两个中间产物的展示层。HTML 仅是审阅/编辑层，最终仍写 `.md` 文件供 `novel-write` 使用。

## 设计

- **故事梗概**：因果链流程图，每段一个节点，垂直排列，箭头连接因果关系
- **场景纲要**：纵向场景卡片，密度色标（高=红/中=琥珀/低=绿），卡片间箭头
- **编辑**：所有文本区域 `contenteditable`，用户直接在浏览器中修改
- **复制回 Claude Code**：底部固定操作栏，三个按钮（复制梗概/复制大纲/复制全部），生成 Markdown 写入剪贴板
- **模板**：`templates/plan-viewer.html`，内嵌 CSS + JS，无外部依赖，通过 `PLAN_DATA` JSON 注入内容

## 流程变更

原：步骤4 写 故事梗概.md → 步骤8 写 大纲.md
新：步骤4 暂存 → 步骤8 确认暂存 → 步骤9 生成 HTML → 用户审阅编辑复制 → 写入 .md 文件

## 下游影响

无。`novel-write` 仍读取 `大纲.md`，格式不变。
