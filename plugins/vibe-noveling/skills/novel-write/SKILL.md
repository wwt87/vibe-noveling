# 章节创作

本章负责三件事：**把大纲变成血肉**（补环境、感官、动作链、台词）；**固定 20 个剧情点为核对骨架**；**双风格写作 + 逐点择优合并**。

## 第一步：读取上下文

### 1.1 读取顺序

1. 本章 `大纲.md` — 提取涉及角色、地点、物品、关键推进链
2. 上一章 `正文.md` — 确认承接状态和章末钩子
3. `memory/past.md` — 最近摘要和当前状态 → **停，不展开**
4. 从大纲提取的角色/地点/物品名，用 novel-knowledge 搜索后读对应 entity 文件
5. `memory/future/20-threads.md` + `90-sync-tracker.md`
6. 当前事件 `memory/future/40-events/` 对应文件（如存在）

设定文件缺失就先补全再写作，不自己编造。

### 1.2 读取 Opus 测试报告

检查 `Opus报告.md` 是否存在：
- 判定 `需先重做大纲` → 立即停止，回 novel-plan --revise
- 判定 `需小修后写` → 提炼为写作约束摘要，写入 `写作约束.md`
- 不存在 → 提示风险，直接按大纲写作

`写作约束.md` 格式（含密度约束）：

```markdown
## 写作约束摘要
- 必须补出：[桥接动作]
- 必须压缩：[信息段]
- 需要加固：[角色声线]
- 禁止重犯：[问题]

## 详略约束
高密度剧情点（{编号}）：200-400字/点，必须含感官层次、在场角色反应、主角内心冲击
低密度剧情点（{编号}）：50-100字/点，只保留动作推进+信息释放
中密度剧情点（{编号}）：120-200字/点，标准处理
```

## 第二步：正文阶段细化剧情点

从大纲的 `可写场景纲要` 细切出恰好 **20 个剧情点**，写入 `剧情点.md`。每个剧情点含：标题、起始状态、核心推进、关系变化、结束压力。

场景纲要是叙述式段落——场景目标、触发事件、冲突落地、转折点、落点变化和细节锚点均嵌入散文中，从散文中提取和理解为 20 个剧情点，字段标签不再出现。

密度继承：每个剧情点继承所属场景块的密度标记（高/中/低），写入剧情点条目：
- **高**：200-400 字/点，必须含感官层次、在场角色反应、主角内心冲击
- **中**：120-200 字/点，标准处理
- **低**：50-100 字/点，只保留动作推进和信息释放，禁止环境描写、配角心理、微动作、过场废话

20 个剧情点不回写 outline，但在所有中间稿和最终稿中显性保留标题。

以下情况必须退回 novel-plan：需新增/删除整段核心推进、需重排关键剧情顺序、需改动章节收尾目标、需整体挪动伏笔处理位置。

## 第三步：合并配置确认

检查 `合并配置.md` 是否存在。不存在则用 AskUserQuestion 询问本章合并偏好（风格映射、基调倾向）：

```markdown
# 合并配置
## 风格偏好
| 剧情点特征 | 优先风格 | 理由 |
|-----------|---------|------|
| 战斗、正面冲击 | 会说话的肘子 | ... |
| 试探、布局 | 大仲马 | ... |

## 全局规则覆盖
<!-- 追加额外合并约束 -->

## 章节基调
- 本章气质倾向：...
- 本章应避免的风格：...
```

用户选择"用默认规则"或跳过则凭创作直觉择优选择。

## 第四步：双风格写作 + 逐点择优合并

### 4.1 双风格写作

主 session 依次写两个风格版本：
1. **会说话的肘子** — 现代快节奏、临场决断、干净利落的冲突推进
2. **大仲马** — 精密布局、戏剧交锋和信息差

写作前读对应 agent 文档（`agents/writer-zhouzi.md` / `agents/writer-dazhongma.md`）作为风格指导。

### 4.1.1 写作前必读

动笔前先读 `references/ai-smell-checklist.md`，将其中 10 项检测标准作为写作约束遵守。核心原则：用动作和感官传递情绪，不用标签直给；对话有刺不协商；修饰具体不空洞，少用比喻；节奏有张弛不平均；细节必须承担叙事功能；句式避AI腔；不水字数；不虚假拔高；不带交流口吻；不翻译腔。写出来的东西尽量一遍过检测，不要留到第六步再修。

### 4.1.2 双版本写作

字数目标 2500-4000，按 20 个剧情点顺序写，标题格式 `【剧情点01：标题】`。两个版本依次完成，不做并发。

### 4.2 逐点择优合并

按 20 个剧情点逐点择优合并为最终稿。硬约束：
1. 每个剧情点只选 1 个主来源版本
2. 最多做必要局部修字，不从其他版本拼句
3. 最终稿不保留剧情点标题和来源标记，正文为连续纯文字

合并优先级：章节 `合并配置.md` 的风格偏好 > 创作直觉。合并后检查：切换连贯性。剧情点之间用自然段落过渡，不出现 `【剧情点` 开头的标题行。

## 第五步：一致性校验

对最终稿调用 `consistency-guard`，重点检查：
- 合并是否引入新设定冲突
- 称谓、时间线、认知是否一致
- 不同来源切换后角色语气和场景因果是否断裂

只做局部修补，不借一致性修补名义重做结构。

## 第六步：AI 味检测与文风复检

主 session 直接执行。读取 `references/ai-smell-checklist.md`，按其中检测项逐项复检正文，每轮只检一项并修复该项暴露的问题。

## 第七步：正文润色

AI 味检测通过后，做一轮独立润色：
- 物品描写：避免"普通、简单、破旧"这类直白词，换成更有质感、更有层次的表达
- 环境描写：补氛围、光泽、纹理、层次和压迫感
- 人物描写：用更有风骨、气势、锋芒和仪态感的词汇
- 行为描写：改成带武侠风味的动作表达
- 润色后句子仍服务主角推进，不因追求华丽词汇让动作发虚、信息发糊

## 第八步：仿人工增强

润色完成后，统计正文 7 类虚词密度（动态助词/结构助词/语气词/介词/副词/判断动词/连词），与手工精修基准对比。基准密度详见 `references/ai-smell-checklist.md` 末尾。

对低于基准 90% 的虚词类别做定向补插，对高于基准 110% 的做定向缩减。补插/缩减用 SSoT（在心里生成随机字符串，用其片段决定操作位置）：
- 补插：在合适位置加虚词（"了""着""的""地""吧"等），允许改写句式引入介词/判断动词/连词
- 缩减：删除冗余程度副词、多余语气词、非必需动态/结构助词，允许改写句式简化介词/判断动词/连词
- 额外：约每 8-10 个句号删 1 个

跳过空行、分隔行。不改事实信息和动作因果。

## 输出

章节目录结构：
```
chapters/vol-{v}/ch-{c}/
├── 大纲.md
├── 剧情点.md        ← 第二步写入
├── 写作约束.md      ← 第一步写入
├── 合并配置.md      ← 第三步写入(可选)
├── 会说话的肘子.md   ← 第四步写入
├── 大仲马.md        ← 第四步写入
└── 正文.md          ← 合并+校验+润色+增强后写入
```

交付后提示用户：标记需修改位置后运行 novel-revise，或确认无需返修后运行 novel-sync。

## 禁止

- 擅自改变章节收尾目标或关键推进链
- 忽略 `剧情思路卡` 的本章边界与核心兑现约束
- 把大纲重新解释成另一个不同的大纲
- 忽略设定缺失自己编造
- 结构问题不上正文阶段硬修——回 novel-plan
- 借一致性修补名义重做结构
